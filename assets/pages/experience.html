Background

My end goal and reason for being a programmer and programming ingeneral is so I can build games. Of course as a kid, I played many games, the main inspirations for my love for them being Minecraft, and the franchises being Age of Empires, SimCity, Elder Scrolls, and Fallout. Since I was 12, I have been developing almost too many unfinished games, due to my ever-changing idea of perfection and my knowledge of syntax getting better over the years. It's been a long process, but I know that some failure and hard-learning lead to the best results in the end.

One of the games I have been working off and on with, constantly rewriting the code and gameplay would be my game Victorum (used to be known as Colonial Warfare.) It's a historical 4X / simulation game about the Age of Colonialism. It begins in the Age of Discovery and ends at the end of the Age of Imperialism. That's my inner-geek coming out though, which if you get to know me, will come out every once in a while. At the moment, this game is being built with Node.js, ElectronJS, JavaScript, HTML, and SASS. The library used to display 3D objects is a proprietary game engine I built that utilizes Three.js.

This game is still a work-in-progress and probably won't be completed for a long time.

WebGL / Three.js

Over the last 2 years, I have been learing about how to implement 3D into the web. Since I was using JavaScript, I needed to find a library that had an easy-to-use API and dedicated community. This way if I ran into an issue, I could get a decent amount of useful help from others. I was originally going to use BabylonJS, but it seemed that Three.js was easier to use out of the box. That became the catalyst for my love and interest in Three.js.

Initially I wanted to use it for my games, and I still do, but I realized it was capable of so much. I started to learn how to create procedurally-generated terrain and landscapes, which were so mezmorizing to look at when I was complete with them. I even posted it to the Three.js Showcase and Mr. Doob, the creator of the library, liked it himself. You can view my showcase here, although the link to the demo doesn't work anymore.

My knowledge since then has become immense. With my understanding of WebGL, Three.js, and ECS Architecture, I have built my own game engine. You can see this on my Projects page.

Node.js / NPM

When I wanted start building games, I also wanted them to have online-connectivity. Since I was already invested in JavaScript, I was looking for something that would enable my JavaScript apps to connect to a server. Low and behold, I found Node.js. Not only was it perfectly integrated with JavaScript, it also has many libraries and NPM packages to make building my apps easier. Since it was also the most modern way to go, I decided to use that to accomplish my goals.

With learning Node.js, I learned how to use lots of packages such as Socket.io, learned how to communicate with the local file-system, and much more related things. Of course in the process, I have created many of my own NPM Packages which include and Entity Component System, the Dimensional Engine, some app utilities to make my development easier, and a simple NPM Package hierarchy builder. A few of my non-depreciated packages can be found under the JavaScript section on my Projects page.

Along the line, I learned how to use and implement ElectronJS for building multi-platform apps. It's a perfect solution for doing so compared to the dystopian wasteland of outdated options out there.

Frameworks

To be honest, I don't like using external frameworks unless I built them. Sometimes working with the ones out there can be really clunky and inefficient for my projects. While I do prefer to develop and interact with my own in-house frameworks, I realize that sometimes, especially for bigger team projects, that it is good to use them. That being said, I am very familiar with React and its "native" brother. I am considering looking into other frameworks, but react seems to be the most widely used one for most teams.

The biggest and most relevant project of mine that is currently being built with React is my game engine, the Dimensional Engine, which you have probably seen mentioned a few times throughout your reading now.
